What

The project implements a queue system with a server and some client apis.

The server is a mongrel web server that holds REST-named queues. You can GET, POST, DELETE, HEAD queue messages using the normal HTTP requests. The system itself uses a configuration file (YAML) to setup queues at startup or even at runtime. The queue implementations can be changed using an easy plugin system right from your project directory.

For an simple administration and try out of the system, FMQ has an integrated ajax based web interface.

The client apis are implemented using the HTTP protocol, so that you can use even curl to receive messages. Ruby is implemented right now, other languages will follow.

The queue itself is an URL like http://localhost:5884/myQueueName/ or http://localhost:5884/myApplication/myQueueName/. If you do a GET request to this url with a web browser you will receive one message from the queue.

Features

  • FIFO message store
  • easy setup and maintainance of system
  • using http for communication
  • changeable queue implementation
  • ruby client lib
  • simple ajax admin interface
  • queues to return same file every time (debugging)
  • queues to forward messages to several queues
  • queues to store data in a FIFO in memory (for hith performance)

Installing

sudo gem install fmq

First steps

After installing the gem you can start by creating a project:

fmq create my_project_name
next step is to change to the folder and start the FMQ server: 
cd my_project_name
fmq

The server will start and host a admin interface on http://localhost:5884/admin/index.

Git, patches, forks and all that stuff

If you want to clone the git repositiory use the command below. Patches, forks and feature requests are welcome, please send them to fmq-3z (at) gmx.net.

git clone git://github.com/threez/fmq.git

Build and test instructions

cd fmq
rake test
rake install_gem

Loadtest

To check the performance we have done a load test. The system under test was a 2 GB dual-core intel 1.8 Ghz with 100 Mbit/s LAN and Ruby 1.8.6. In the test we simulate 20 applications (with 20 threads). Each of this applications did 2 POSTs and 1 GET request for 30min. The POST requests where about 1KB.

This are the results (from 2008-06-10; MongrelHandler):

Test duration: 30:01 min
Average response time: 36 ms
Passed Transactions: 862203
Failed (because of connection timeout): 285
Transactions per second: 476,1
Transactions per minute: 28.749,60
Transactions per hour: 1.724.976
